C# NORMAS DE CODIFICACIÓN Y CONVENCIONES DE NOMENCLATURA

NORMAS GENERALES: Toda la codificación realizada es decir creación de formularios, clases, propiedades, métodos, etc, es en inglés.

CLASES:Nombramiento en singular

METODOS:Debe iniciar con un verbo (acción).

PROPIEDADES:Nombramiento en singular, a menos que sea una colección.

INSTANCIAS A CLASES:En lo posible usar el patrón de singleton, y debe quedar como una propiedad de la clase desde la cual está siendo invocada.

HACER Utilice Pascal Case para nombres de clases y métodos 
      
      public class ClientActivity
      {{ '{' }}
          public void ClearStatistics()
          {{ '{' }}
              // ..
          {{'}'}}

          public void CalculateStatistics()
          {{ '{' }}
              // ..
          {{'}'}}
      {{'}'}}
HACER Utilice camelCasing para argumentos de método y variables locales 
      
      public class ClientActivity
      {{ '{' }}
          public void Add(LogEvent logEvent)
          {{ '{' }}
              int itemCount = logEvent.Items.Count;
          {{'}'}}
      {{'}'}}
NO HACER Utilizar la notación húngara o cualquier otra identificación de tipo en los identificadores 
      
      /// Correct
      int counter;
      string name;

      /// Avoid
      int iCounter;
      string strName;
NO HACER Usar mayúsculas para constantes o variables de sólo lectura 
      
      /// Correct
      public static const string ShippingType = "DropShip";

      /// Avoid
      public static const string SHIPPINGTYPE = "DropShip";
EVITAR Usar las abreviaturas. Excepciones: abreviaturas comúnmente usadas como nombres, como Id, Xml, Ftp, Uri 
      
      /// Correct
      UserGroup userGroup;
      Assignment assignment;

      /// Avoid
      UserGroup usrGrp;;
      Assignment empAssignment;;

      /// Exception
      CustomerId iCounter;
      XmlDocument xmlDocument;
      FtpHelper ftpHelper;
      UriPart uriPart;
NO HACER Utilizar raya al piso en los identificadores. Excepción: puede usarse variables estáticas privadas. 
      
      /// Correct
      public DateTime clientAppointment;
      public TimeSpan timeLeft;

      /// Avoid
      public DateTime client_Appointment;
      public TimeSpan time_Left;

      /// Exception
      private DateTime _registrationDate;
HACER Utilizar nombres de tipo predefinidos en lugar de nombres de tipo de sistema como Int16, Single, UInt64, etc. 
      
      /// Correct
      string firstName;
      int lastIndex;
      bool isSaved;

      /// Avoid
      String firstName;
      Int32 lastIndex;
      Boolean isSaved;
HACER Utilizar sustantivos para nombrar una clase. 
      
      public class Employee
      {{ '{' }}

      {{'}'}}

      public class BusinessLocation
      {{ '{' }}

      {{'}'}}
HACER Utilizar en las interfaces el prefijo con la letra I. Los nombres de interfaz son sustantivos (frases) o adjetivos. 
      
      public interface IShape
      {{ '{' }}

      {{'}'}}

      public interface IShapeCollection
      {{ '{' }}

      {{'}'}}

      public interface IGroupable
      {{ '{' }}

      {{'}'}}
HACER Declarar todas las variables en la parte superior de la clase. En el siguiente orden, contantes, estáticas y propiedades. 
      
      // Correct
      public class Account
      {{ '{' }}
          public static string  BankName;
          public static decimal  Reserves;

          public string Number {{ '{' }} get; set; {{'}'}}
          protected DateTime DateOpened {{ '{' }} get; set; {{'}'}}
          private DateTime DateClosed {{ '{' }} get; set; {{'}'}}
          public decimal Balance {{ '{' }} get; set; {{'}'}}
      {{'}'}}

      // Constructor
      public Account()
      {{ '{' }}
          // ...
      {{'}'}}
