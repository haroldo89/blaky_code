ANGULAR NORMAS DE CODIFICACIÓN Y CONVENCIONES

Creación de Componente

Para la creación de componentes en el proyecto, debemos ejecutar la consola y acceder a la carpeta del proyecto. A continuación, digite el siguiente comando:

ng generate component carpetaContenedora/nombreDelComponente

forma abreviada

ng g c carpetaContenedora/nombreDelComponente 

    
    ng g c components/login ClientActivity
Creación de Servicios

Para la creación de Servicios en el proyecto, debemos ejecutar la consola y acceder a la carpeta del proyecto. A continuación, digite el siguiente comando:

ng generate service carpetaContenedora/nombreDelComponente

forma abreviada

ng g s carpetaContenedora/nombreDelComponente 

    
    ng g s services/login ClientActivity
Creación de clases

Para la creación de clases en el proyecto, debemos ir a la carpeta donde se almacenaran las clases y crear un archivo con la siguiente estructura:

Nombre: NombreDeLaClase.ts bajo PascalCase

Ejemplo: UserLogged.ts

Dentro del archivo: 


    export class UserLogged {{'{'}}
        id: number;
        name: string;
        username: string;
        password: string;
        securityLevel: number;
    {{'}'}}
Declaración De Variables En Componentes (Archivos Ts)

En los archivos Ts, se deben declarar las variables dentro de la clase del componente y antes del constructor en camelCase como se ve a continuación 

    
    export class LoginComponent implements OnInit {{'{'}}
        loginData: string; //bajo camelCase
        constructor() {{'{}'}}
            …
            …
            {{'}'}}
Manejo de Variables de entorno (pruebas y producción)

Dentro de la carpeta scr/enviroment se almacenan 2 archivos uno para pruebas y el de producción, lo cual nos permite manejar constantes diferentes tanto para producción como para pruebas, actualmente se maneja la variable “baseUrl” en la cual se almacena el prefijo de la aplicación, para pruebas es un string vacio (dado que se está trabajando de forma local, en caso de querer trabajar en dos puertos diferentes ejemplo front en 4200 y back en 60583, se debe llenar esta variable de la siguiente manera: http://localhost:60583 ) y para producción es el nombre de la aplicación web en el servidor y se le antepone un '/'.

En enviroments.ts(pruebas desde el mismo puerto) 


    export constenvironment = {{'{'}}
        production: false
        baseUrl: "" /* URL Pruebas */
    {{'}'}}}

En enviroments.ts (pruebas para trabajar desde el puerto 4200 (ng serve)) 


    export const environment = {{'{'}}
        production: false
        baseUrl: "http://localhost:60583" /* URL Desarrollo */
    {{'}'}}

En enviroments.prod.ts (pruebas para trabajar desde el puerto 4200 (ng serve)) 


    export const environment = {{'{'}}
        production: false
        baseUrl: "/OfertaSugeridaWarriors" /* URL Desarrollo */
    {{'}'}}
Compilar angular en pruebas

Desde la consola y estando ubicado en la carpeta front-end del proyecto, ejecute el comando “ng build”, este comando generara los archivos compilados los cuales se almacenaran en la carpeta “Front-End\Scripts\libs”:

Ejemplo: 

    
    ng build

  • a la carpeta 'Front-End\Scripts\libs'.
  • Copie todos los archivos en esta.
  • Pegue los archivos en la carpeta Main del proyecto, reemplazando todos los archivos.

Con lo anterior si se ejecuta el proyecto se tomaran las actualizaciones, en caso de que este no sea el caso, borre la cache y recargue la página.

Compilar angular en producción o servidor de pruebas

Desde la consola y estando ubicado en la carpeta front-end del proyecto, ejecute el comando 'ng build --prod --base-href /nombre del proyecto en el servidor/', este comando generara los archivos compilados los cuales se almacenaran en la carpeta “Front-End\Scripts\libs”:

Ejemplo: 

    
    ng build --prod --base-href  /OfertaSugeridaWarriors/
Auto cargar cambios

Esta funcionalidad permite que cuando se actualice el proyecto en el servidor este sea actualizado en el cache del usuario final, luego de cargar la pagina y para esto se debe correr el siguiente comando luego del ng build y antes de pasar al servidor:

Ejemplo: 

    
    npm run post-build

El script anterior toma el valor del código posterior al nombre main del compilado y lo almacena en el archivo version.json para luego en la carga de los datos si se encuentra que este valor ha cambiado recarga la pagina en su ultima versión(actualiza cache).

Descargar librerías del proyecto

Para lo anterior se recuerda que deben haber instalado npm, nodejs y angular cli, adicionalmente los comandos npm no son ejecutables desde Odin se debe hacer desde la red de invitados. Tambien se debe tener en cuenta que para esta instalación el proyecto, no debe contar con la carpeta node_modules, ni tampoco con el archivo package.lock (si están deben ser eliminados).

Desde la consola y estando ubicado en la carpeta front-end del proyecto, ejecute el comando 'npm install', este comando toma todas las librerías que requiere el proyecto y las instala en la carpeta node modules.

Ejemplo: 

    
    npm install
